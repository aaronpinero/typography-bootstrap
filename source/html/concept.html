Concept

There are three main design enhancements that are applied to Bootstrap with the addition of the Tyfy framework:

  • a modular typographic scale;
  • a consistent vertical grid; and
  • intelligent margins.

format_size

Modular Scale

Tim Brown, head of Typography at Adobe Typekit, has this to say about modular scales:

By basing the dimensions of our compositions on values from modular scales, we can achieve a visual harmony not found in layouts that use arbitrary, conventional, or easily divisible numbers.

More simply, from Robert Bringhurst:

A modular scale, like a musical scale, is a prearranged set of harmonious proportions.

For typography, employing a modular scale for type sizes results in a series of type sizes that are related by a common mathematical ratio.

Tyfy provides a ratio variable and a custom function for calculating type sizes for paragraphs, headings, and other elements that conform to a modular scale. The Bootstrap font size variables in this theme have been modified using the scale() function. Because all the font sizes are based on the ratio variable and the base font size, modifying either variable will quickly change all of the theme's font sizes.

Vertical Grid

In many articles, this is also called a baseline grid because it is the grid on which the baseline of your text will run. Establishing a vertical or baseline grid provides a often unnoticed enhancement to the design. Robert Bringhurst likens a baseline grid to the rhythm that provides a foundation for musical composition.

Bootstrap does not have a well established vertical grid. There is a spacer variable that is used to provide padding and margin on some elements, but there are odd omissions. In addition, the line height for text is based on a percentage of the current font size instead of on a grid measure.

Tyfy provides a grid unit SASS variable and two custom functions that facilitate the application of a baseline grid. The grid() function eliminates the need to calculate fractions or multiples of the baseline grid unit. The leading() function calculates a line height for a given font size as a function of multiples of the grid unit. Together, these functions eliminate much of the work of using a baseline grid and make it easier to employ. The Tyfy Bootstrap theme changes a number of the default Bootstrap element line-height, margin, and padding values so that they conform to the grid. As a result, a change to the value of the grid unit variable quickly changes the entire theme.

vertical_align_center

Intelligent Margins

The default HTML behavior for elements like headings, paragraphs, and lists is to provide margin above and below the element. Bootstrap and other frameworks often modify these margins, but perpetuate the practice of providing arbitrary margins around elements. Unfortunately, these extra margins cause unexpected or unwanted collisions at the beginning or end of a containing element, depending on whether or not the containing element has its own margin or padding.

Margins around elements are unnecessary.
Margins between elements are what is needed.

The Tyfy framework uses adjacent sibling selectors (i.e. .class + .class) to assign margins between elements instead of assigning blanket margins to everything. This avoids the problem of having margins at the start and end of a containing element that must then be accounted for.
